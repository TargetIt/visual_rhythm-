📖 How Naive SGEMM Works

Each thread computes one element of C[i][j] by iterating over the shared K-dimension:

C[i][j] = Σ (k=0..K-1) A[i][k] × B[k][j]

This kernel reads K elements from a row of A and K elements from a column of B for every output element, with no data reuse across threads — making it severely memory-bound.

The arithmetic intensity is only 2 / (2×4) = 0.25 FLOP/Byte, far below what GPUs need to be compute-bound.
